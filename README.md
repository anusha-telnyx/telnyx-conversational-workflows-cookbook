# Telnyx Conversational Workflows Cookbook

Build an inbound Telnyx Conversational Workflow that collects auto insurance first notice of loss details, calls live claim-intake tools, flags urgent cases, and exports structured conversation insights for review.

This repo is a runnable cookbook for designing, testing, and presenting a node-based Telnyx Conversational Workflow.

For a deeper walkthrough of the call flow, workflow graph, tool calls, fallback paths, and insights mapping, see `docs/how-it-works.md`.

## What You Are Building

```txt
inbound call
-> conversational workflow greeting
-> claim intent classification
-> safety triage
-> caller and loss detail collection
-> claim intake tool call
-> optional priority follow-up
-> structured conversation insights
```

## What is included

- `prompts/assistant_prompt.txt`: optional legacy assistant prompt, not the workflow source of truth.
- `prompts/greeting.txt`: workflow first message text.
- `workflows/auto_claim_intake_workflow.json`: source of truth for workflow nodes, branches, variables, and tool calls.
- `workflows/portal_build_guide.md`: step-by-step mapping for building the Telnyx Conversational Workflow in the Portal.
- `faq/faq_template.md`: approved claims FAQ for the workflow knowledge source if enabled.
- `api/mock_claim_api.mjs`: dependency-light mock claim API for live Telnyx tool calls.
- `automation/tool_contracts.md`: request and response contracts for the live tools.
- `schemas/conversation_insights_schema.json`: target structure for Telnyx conversation insights.
- `automation/zapier_make_mapping.md`: field mapping for Telnyx insights to Google Sheets or another tracker.
- `tests/manual_test_plan.md`: call scenarios for acceptance testing.
- `templates/prd_template.md`: contributor-facing PRD template for defining requirements, artifacts, acceptance criteria, and review signoff before workflow changes.
- `outputs/auto_claim_intake_summaries.csv`: claim summary template generated by `npm run build:workbook`.
- `outputs/setup_checklist.csv`: setup checklist template generated by `npm run build:workbook`.

## Requirements

You need:

- Node.js 20 or newer.
- A Telnyx account.
- An inbound voice-capable Telnyx phone number.
- Access to Telnyx Conversational Workflows.
- A public HTTPS URL that can forward to the local mock API while testing.

## Install

Install dependencies if needed:

```bash
npm install
```

Copy the example environment file:

```bash
cp .env.example .env
```

For local testing, set:

```bash
CLAIM_TOOL_SECRET=dev-secret
PORT=8787
```

## Local mock API

Start the mock claim API:

```bash
CLAIM_TOOL_SECRET=dev-secret npm start
```

Health check:

```bash
curl -s http://localhost:8787/health
```

Expose the API to Telnyx with a public URL before configuring live tools. Use the same `x-tool-secret` header in each Telnyx webhook tool.

## Telnyx setup

### Conversational Workflow setup

1. Open the Telnyx Portal.
2. Go to Conversational Workflows.
3. Create a workflow named `auto claim intake conversational workflow`.
4. Build the node graph from `workflows/auto_claim_intake_workflow.json`.
5. Use `workflows/portal_build_guide.md` to map each node, branch, variable, and tool call.
6. Configure voice as `voice ultra katie`.
7. Configure live tool nodes against the public mock API host.
8. Assign the inbound Telnyx phone number to the workflow.
9. Run the scenarios in `tests/manual_test_plan.md`.

### Assistant setup, optional legacy path

1. Create or log into a Telnyx account.
2. Buy an inbound voice-capable Telnyx phone number.
3. Open the AI Assistants area in the Telnyx Portal.
4. Create a new assistant named `auto claim intake agent`.
5. Paste `prompts/assistant_prompt.txt` into the assistant instructions.
6. Paste `prompts/greeting.txt` into the greeting field.
7. Configure voice as `voice ultra katie`.
8. Enable noise suppression if available.
9. Upload `faq/faq_template.md` as the knowledge base after business approval.
10. Configure three live webhook tools using `automation/tool_contracts.md`.
11. Assign the Telnyx phone number to the assistant for inbound calls.
12. Configure conversation insights using `schemas/conversation_insights_schema.json`.
13. Configure the insights webhook to Zapier, Make, or a claims reporting endpoint.
14. Import the generated workbook into Google Sheets if using sheet-based QA.
15. Run the scenarios in `tests/manual_test_plan.md`.

## Workflow tools

Configure these Telnyx workflow tool nodes against the public API host:

- `create_claim_intake`: `POST /tools/create-claim-intake`
- `log_claim_intake_fallback`: `POST /tools/log-claim-intake-fallback`
- `flag_priority_follow_up`: `POST /tools/flag-priority-follow-up`

The mock API stores no persistent data. It validates required inputs and returns realistic reference IDs so the conversation workflow and tool behavior can be tested before connecting a real claims system.

## Acceptance criteria

- The workflow confirms the caller is reporting an auto claim.
- The workflow triages safety and urgent cases early.
- The workflow collects minimum claim details before calling `create_claim_intake`.
- Every completed call gets either a `claim_intake_id` or a `fallback_reference_id`.
- Urgent cases set `priority_flag` and create a priority follow-up task.
- The workflow does not promise coverage, liability, payment, rental, towing, repair authorization, medical advice, legal advice, or settlement timing.
- Conversation insights match the claim tool payload and Google Sheet mapping.
